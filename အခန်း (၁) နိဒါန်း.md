# **၁။ နိဒါန်း**

Ngspice သည် nonlinear နှင့် linear ခွဲခြမ်းစိတ်ဖြာမှုများအတွက် ယေဘူယျသုံး ဆားကစ် သရုပ်ဖော် ပရိုဂရမ်တစ်ခု ဖြစ်သည်။ ဆားကစ်များတွင် ရီဇစ္စတာ (resistor)၊ ကာပါစီတာ (capacitor)၊ အင်ဒက်တာ (inductor)၊ အပြန်အလှန် အင်ဒက်တာ (mutual inductor)၊ သီးခြား သို့မဟုတ် မှီခိုသော ဗို့အား နှင့် လျှပ်စီးကြောင်း ရင်းမြစ် (independent or dependent voltage and current source)၊ ဆုံးရှုံးမှုမရှိ နှင့် ဆုံးရှုံးမှုရှိသော ထုတ်လွှင့်လိုင်း (loss-less and lossy transmission line)၊ ခလုတ် (switch)၊ ညီညီ ဖြန့်ဝေထားသော RC လိုင်း (uniform distributed RC line) နှင့် အသုံးအများဆုံး တစ်ပိုင်းလျှပ်ကူး ပစ္စည်း (semiconductor device) ငါးမျိုး- ဒိုင်အိုဒ် (diode)၊ BJT၊ JFET၊ MESFET နှင့် MOSFET တို့ ပါဝင်နိုင်သည်။

Ngspice ကို အသုံးပြုရန် အသုံးအများဆုံးနည်းလမ်းမှာ OS command prompt မှ စတင်၍ netlist ဖိုင် (ဆားကစ်တစ်ခု၏ အဓိပ္ပါယ်ဖွင့်ဆိုချက်) အမည်ကို ပေးပို့ခြင်း ဖြစ်သည်။ ဤလက်စွဲ၏ အဓိကအပိုင်းမှာ ထိုကဲ့သို့သော ဖိုင်များ၏ ဖော်ပြချက်ဖြစ်သည်။ စတင်ခြင်းဆိုင်ရာ ရွေးချယ်စရာများ အပြည့်အစုံအတွက် အခန်း ၁၂ ကို ကြည့်ပါ။ ထည့်သွင်းဖိုင်များတွင် Ngspice ၏ command language (၁၃) ဖြင့် ရေးသားထားသော script များလည်း ပါဝင်နိုင်သည်။ အပြန်အလှန်သုံး အသုံးပြုသူ မျက်နှာပြင်များ (interactive user interfaces) ကို အခန်း ၁၄ တွင် ဖော်ပြထားသည်။

Ngspice ကို မည်သို့စတင်အသုံးပြုရမည်နှင့် ပတ်သက်သော မိတ်ဆက်မှတ်စုအချို့ကို အခန်း ၁၇ တွင် တွေ့နိုင်သည်။

Ngspice သည် Spice3f5၊ နောက်ဆုံး Berkeley ၏ Spice3 simulator မိသားစု ထုတ်ဝေမှု၏ အဆင့်မြှင့်တင်မှုတစ်ခု ဖြစ်သည်။ Ngspice သည် ရှိပြီးသား Spice3f5 သို့ အင်္ဂါရပ်အသစ်များ ထည့်သွင်းရန်နှင့် ၎င်း၏ bugs များကို ပြင်ဆင်ရန် ဖွံ့ဖြိုးတိုးတက်နေသည်။ ဆားကစ် simulator ကဲ့သို့သော ရှုပ်ထွေးသည့် ဆော့ဖ်ဝဲလ်ကို မြှင့်တင်ခြင်းသည် အလွန်ခက်ခဲသော အလုပ်ဖြစ်ပြီး၊ တိုးတက်မှုအချို့ ပြုလုပ်ထားသော်လည်း အလုပ်အများစုမှာ bug ပြင်ဆင်ခြင်းနှင့် code ပြန်လည်ဖွဲ့စည်းခြင်း (refactoring) ပေါ်တွင် ဖြစ်သည်။

Ngspice တွင် တစ်ပိုင်းလျှပ်ကူးပစ္စည်းများအတွက် built-in မော်ဒယ်များပါရှိပြီး၊ အသုံးပြုသူသည် သက်ဆိုင်ရာ မော်ဒယ် ပါရာမီတာ တန်ဖိုးများကိုသာ သတ်မှတ်ပေးရန် လိုအပ်သည်။

Ngspice သည် ရောနှောအဆင့် သရုပ်ဖော်ခြင်း (mixed-level simulation) ကို ထောက်ပံ့ပေးပြီး နည်းပညာဆိုင်ရာ ပါရာမီတာများ (technology parameters) နှင့် ဆားကစ် စွမ်းဆောင်ရည် (circuit performance) အကြား တိုက်ရိုက်ချိတ်ဆက်မှုကို ပေးပါသည်။ ရောနှောအဆင့် ဆားကစ်နှင့် ပစ္စည်းကိရိယာ simulator သည် ဆားကစ်အတွင်းရှိ အရေးပါသော ပစ္စည်းကိရိယာများကို ဂဏန်းနည်းဖြင့် မော်ဒယ်လုပ်ခြင်းဖြင့် သီးသန့် ဆားကစ် သို့မဟုတ် ပစ္စည်းကိရိယာ simulator ထက် ပိုမိုတိကျသော သရုပ်ဖော်မှုကို ပေးစွမ်းနိုင်သည်။ အခြားသော ပစ္စည်းကိရိယာအားလုံးအတွက် Compact model များကို အသုံးပြုနိုင်သည်။ Ngspice သို့ mixed-level extensions သည် CIDER ဖြစ်ပြီး၊ ၎င်းသည် ngspice code အတွင်းသို့ ပေါင်းစည်းထားသော ရောနှောအဆင့် ဆားကစ်နှင့် ပစ္စည်းကိရိယာ simulator တစ်ခုဖြစ်သည်။

Ngspice သည် XSPICE code ပေါင်းစည်းမှုမှတစ်ဆင့် ရောနှောအချက်ပြ သရုပ်ဖော်ခြင်း (mixed-signal simulation) ကို ထောက်ပံ့ပေးသည်။ GeorgiaTech မှ Spice3C1 သို့ extension တစ်ခုအဖြစ် တီထွင်ခဲ့သော XSPICE ဆော့ဖ်ဝဲလ်ကို 'board' level နှင့် ရောနှောအချက်ပြ သရုပ်ဖော်မှု ပေးစွမ်းရန် ngspice သို့ မြှင့်တင်၍ သယ်ဆောင်လာခဲ့သည်။ Verilator သို့မဟုတ် Icarus Verilog ဖြင့် compile လုပ်ထားသော Digital Verilog module များကို ချိတ်ဆက်နိုင်သည်။ Pipe များမှတစ်ဆင့် (C ဖြင့် code ရေးထားသော) process များနှင့် ဆက်သွယ်မှု ထူထောင်နိုင်သည်။

XSPICE extension သည် သီးသန့် digital သရုပ်ဖော်ခြင်းကိုလည်း လုပ်ဆောင်နိုင်စေသည်။

Ngspice သို့ ပစ္စည်းကိရိယာအသစ်များကို နည်းလမ်းများစွာဖြင့် ထည့်သွင်းနိုင်သည်- behavioral B- , E- သို့မဟုတ် G- source များ၊ C-ကဲ့သို့သော ပစ္စည်းကိရိယာ coding အတွက် XSPICE code-model interface နှင့် OpenVAF ဖြင့် compile လုပ်သည့်အခါ OSDI interface မှတစ်ဆင့် Verilog-A မော်ဒယ်များ။

နောက်ဆုံးအနေဖြင့်၊ သေးငယ်သော bug မြောက်မြားစွာကို ရှာဖွေတွေ့ရှိပြီး ပြင်ဆင်ခဲ့ပြီး၊ ပရိုဂရမ်ကို ပိုမိုကျယ်ပြန့်သော ကွန်ပျူတာပလက်ဖောင်းများသို့ သယ်ဆောင်လာခဲ့သည်။

## ၁.၁ သရုပ်ဖော်ခြင်းဆိုင်ရာ အယ်လ်ဂိုရစ်သမ်များ

ကွန်ပျူတာအခြေပြု ဆားကစ် သရုပ်ဖော်ခြင်းကို ဒီဇိုင်နာများ၊ စမ်းသပ်အင်ဂျင်နီယာများနှင့် ရုပ်ပိုင်းဆိုင်ရာ ဆားကစ်ကို မစစ်ဆေးဘဲ ဒီဇိုင်းတစ်ခု၏ လည်ပတ်မှုကို ခွဲခြမ်းစိတ်ဖြာလိုသူများ က ကိရိယာတစ်ခုအဖြစ် မကြာခဏ အသုံးပြုကြသည်။ သရုပ်ဖော်ခြင်းက သင့်အား ဆားကစ် element များစွာ၏ ပါရာမီတာများကို လျင်မြန်စွာ ပြောင်းလဲနိုင်စေပြီး ၎င်းတို့က ဆားကစ် တုံ့ပြန်မှုအပေါ် မည်သို့သက်ရောက်သည်ကို ဆုံးဖြတ်နိုင်စေသည်။ ရုပ်ပိုင်းဆိုင်ရာ ဆားကစ်တစ်ခုတွင် ဤပါရာမီတာများကို ပြောင်းလဲရန်မှာ မကြာခဏ ခက်ခဲသည် သို့မဟုတ် မဖြစ်နိုင်ပေ။

သို့သော် လက်တွေ့ဖြစ်ရန်၊ simulator တစ်ခုသည် ကျိုးကြောင်းဆီလျော်သော အချိန်အတွင်း လုပ်ဆောင်ရမည်။ ထိရောက်သော လုပ်ဆောင်မှု၏ သော့ချက်မှာ ပေးထားသော ပြဿနာတစ်ခုအတွက် သင့်လျော်သော modeling abstraction အဆင့်ကို ရွေးချယ်ခြင်းဖြစ်သည်။ ပေးထားသော modeling abstraction ကို ထောက်ပံ့ရန်၊ simulator သည် သင့်လျော်သော algorithms များကို ပေးအပ်ရမည်။

သမိုင်းကြောင်းအရ၊ circuit simulator များသည် analog simulation algorithm တစ်ခု သို့မဟုတ် digital simulation algorithm တစ်ခုကို ထောက်ပံ့ပေးခဲ့သည်။ Ngspice သည် XSPICE framework ကို အမွေဆက်ခံပြီး analog နှင့် digital algorithms နှစ်မျိုးလုံးကို ထောက်ပံ့ကာ 'mixed-mode' simulator တစ်ခု ဖြစ်သည်။

### ၁.၁.၁ Analog သရုပ်ဖော်ခြင်း

Analog simulation သည် စဉ်ဆက်မပြတ် အချိန် သို့မဟုတ် ကြိမ်နှုန်း ကာလအတွင်း ဆားကစ်တစ်ခု၏ linear နှင့် non-linear အပြုအမူအပေါ် အာရုံစိုက်သည်။ ဆားကစ် တုံ့ပြန်မှုကို အဖြေသည် တည်ငြိမ်သောတန်ဖိုးသို့ ရောက်ရှိကြောင်း နှင့် ပေါင်းစည်းမှုများ၏ ဂဏန်းဆိုင်ရာ အနီးစပ်ဆုံးခန့်မှန်းချက်များသည် လုံလောက်စွာတိကျကြောင်း သေချာစေရန် ရွေးချယ်ထားသော အချိန်အဆင့်များတွင် ဆားကစ်အတွက် Kirchhoff's Laws ကို အကြိမ်ကြိမ် ဖြေရှင်းခြင်းဖြင့် ရယူသည်။ Kirchhoff's laws များသည် တစ်ပြိုင်နက်ညီမျှခြင်းများ အစုတစ်ခုကို ဖွဲ့စည်းသောကြောင့်၊ simulator သည် အချိန်အမှတ်တစ်ခုစီတွင် ညီမျှခြင်းများ၏ matrix တစ်ခုကို ဖြေရှင်းခြင်းဖြင့် လည်ပတ်သည်။ ဤ matrix လုပ်ဆောင်ခြင်းသည် ယေဘူယျအားဖြင့် digital circuit simulator များနှင့် နှိုင်းယှဉ်ပါက သရုပ်ဖော်ချိန် ပိုမိုနှေးကွေးစေသည်။

ဆားကစ်တစ်ခု၏ တုံ့ပြန်မှုသည် အသုံးပြုထားသော source များ၏ function တစ်ခုဖြစ်သည်။ Ngspice သည် DC၊ sine-wave နှင့် pulse အပါအဝင် source အမျိုးအစားများစွာကို ပေးဆောင်သည်။ Source များကို သတ်မှတ်ခြင်းအပြင်၊ အသုံးပြုသူသည် လုပ်ဆောင်ရမည့် simulation အမျိုးအစားကို သတ်မှတ်ရမည်။ ၎င်းကို 'analysis mode' ဟု ခေါ်သည်။ Analysis mode များတွင် DC analysis၊ AC analysis နှင့် transient analysis တို့ ပါဝင်သည်။ DC analysis အတွက်၊ reactive element များ၏ အချိန်နှင့်အမျှ ပြောင်းလဲနေသော အပြုအမူကို လျစ်လျူရှုပြီး simulator သည် ဆားကစ်၏ DC solution ကို တွက်ချက်သည်။ Swept DC analysis ကိုလည်း ngspice ဖြင့် လုပ်ဆောင်နိုင်သည်။ ၎င်းမှာ input source များအတွက် DC အဆင့်များ အကွာအဝေးတစ်ခုအတွင်း DC analysis ကို ထပ်တလဲလဲ အသုံးချခြင်းသာ ဖြစ်သည်။ AC analysis အတွက်၊ simulator သည် reactive element များအပါအဝင် ဆားကစ်၏ တုံ့ပြန်မှုကို ကြိမ်နှုန်းများ အကွာအဝေးတစ်ခုအတွင်း small-signal sinusoidal input များသို့ ဆုံးဖြတ်သည်။ ဤအခြေအနေတွင် simulator output တွင် ကြိမ်နှုန်း၏ function အဖြစ် amplitudes နှင့် phases များ ပါဝင်သည်။ Transient analysis အတွက်၊ reactive element များအပါအဝင် ဆားကစ် တုံ့ပြန်မှုကို အချိန်၏ function အဖြစ် ဆားကစ်၏ အပြုအမူကို တွက်ချက်ရန် ခွဲခြမ်းစိတ်ဖြာသည်။

### ၁.၁.၂ Matrix ဖြေရှင်းကိရိယာများ

Version 42 မှစ၍ ngspice သည် matrix solver နှစ်ခုကို ပေးဆောင်သည်။ Spice3f5 သည် မူလက solver Sparse 1.3 ကို အသုံးပြုခဲ့ပြီး၊ ၎င်းသည် simulation tasks အားလုံးအတွက် ခိုင်မာကြောင်း သက်သေပြထားသည် [26]။ ၎င်းသည် behavioral models များကို သရုပ်ဖော်ရန်အတွက် အထူးသင့်လျော်သည်။ ထောင်ပေါင်းများစွာသော transistor များပါသော ကြီးမားသော ဆားကစ်များ၏ simulation ကို အရှိန်မြှင့်ရန် ရွေးချယ်နိုင်သည့်အနေဖြင့် KLU matrix solver [27, 28] ကို ရွေးချယ်နိုင်သည် (အခန်း ၁၁.၁.၁ ကိုကြည့်ပါ)။

### ၁.၁.၃ Analog သရုပ်ဖော်ခြင်းအတွက် ပစ္စည်းကိရိယာ မော်ဒယ်များ

Bipolar junction transistor အတွက် မော်ဒယ်သုံးမျိုးရှိပြီး၊ အားလုံးသည် Gummel နှင့် Poon တို့၏ integral-charge model ကို အခြေခံထားသည်။ သို့သော် Gummel-Poon parameters များကို မသတ်မှတ်ပါက၊ အခြေခံမော်ဒယ် (BJT) သည် ပိုမိုရိုးရှင်းသော Ebers-Moll model သို့ လျှော့ချသည်။ မည်သည့်ကိစ္စတွင်ဖြစ်စေ၊ မည်သည့်မော်ဒယ်တွင်ဖြစ်စေ၊ charge storage effects၊ ohmic resistances နှင့် current-dependent output conductance တို့ကို ထည့်သွင်းနိုင်သည်။ ဒုတိယ bipolar model BJT2 သည် substrate diode အတွင်းရှိ dc current တွက်ချက်မှုကို ထည့်သွင်းသည်။ တတိယမော်ဒယ် (VBIC) တွင် အဆင့်မြင့် bipolar ပစ္စည်းများအတွက် နောက်ထပ် မြှင့်တင်မှုများ ပါဝင်သည်။

တစ်ပိုင်းလျှပ်ကူး ဒိုင်အိုဒ် မော်ဒယ်ကို junction diodes သို့မဟုတ် Schottky barrier diodes အတွက် အသုံးပြုနိုင်သည်။ JFET အတွက် မော်ဒယ်နှစ်မျိုးရှိသည်- ပထမ (JFET) သည် Shichman နှင့် Hodges တို့၏ မော်ဒယ်ကို အခြေခံပြီး၊ ဒုတိယ (JFET2) သည် Parker-Skellern မော်ဒယ်ကို အခြေခံသည်။ မူလ MOSFET မော်ဒယ်ခြောက်ခုလုံးကို အကောင်အထည်ဖော်ထားသည်- MOS1 ကို square-law I-V characteristic ဖြင့် ဖော်ပြပြီး၊ MOS2 [28] သည် analytical model ဖြစ်ကာ၊ MOS3 [28] သည် semi-empirical model ဖြစ်သည်။ MOS6 [2] သည် short channel region တွင် တိကျသော ရိုးရှင်းသည့် analytic model ဖြစ်ပြီး၊ MOS9 သည် အနည်းငယ် ပြုပြင်ထားသော Level 3 MOSFET model (Philips level 9 နှင့် မရောထွေးပါနှင့်)။ BSIM 1 [3, 4] နှင့် BSIM2 [5] တို့သည် BSIM (Berkeley Short-channel IGFET Model) မော်ဒယ်ဟောင်းများဖြစ်သည်။ MOS2, MOS3, နှင့် BSIM တို့တွင် channel-length modulation, subthreshold conduction, scattering-limited velocity saturation, small-size effects, နှင့် charge controlled capacitances ကဲ့သို့သော ဒုတိယအဆင့် သက်ရောက်မှုများ ပါဝင်သည်။ Submicron ပစ္စည်းများအတွက် မကြာသေးမီက MOS မော်ဒယ်များမှာ BSIM3 (Berkeley BSIM3 web page) နှင့် BSIM4 (Berkeley BSIM4 web page) မော်ဒယ်များဖြစ်သည်။ Silicon-on-insulator MOS transistor များကို BSIMSOI family (Berkeley BSIMSOI web page) မှ SOI မော်ဒယ်များနှင့် STAG [18] မော်ဒယ်တို့ဖြင့် ဖော်ပြသည်။ HFET မော်ဒယ်အနည်းငယ်နှင့် MESA ပစ္စည်းများအတွက် မော်ဒယ်တစ်ခုအတွက် အနည်းငယ် ထောက်ပံ့မှုရှိသည်။ OpenVAF/OSDI interface (အခန်း ၉ ကိုကြည့်ပါ) မှတစ်ဆင့် Verilog-A မော်ဒယ်များကို ရရှိနိုင်ပါသည်။

### ၁.၁.၄ Digital သရုပ်ဖော်ခြင်း

Digital circuit simulation သည် analog circuit simulation နှင့် ကဏ္ဍပေါင်းစုံတွင် ကွဲပြားသည်။ အဓိကကွာခြားချက်မှာ Kirchhoff's laws ၏ အဖြေကို မလိုအပ်ခြင်းဖြစ်သည်။ ထိုအစား၊ simulator သည် node တစ်ခု၏ logic state ပြောင်းလဲမှု ဖြစ်ပေါ်ခြင်း ရှိ/မရှိကိုသာ ဆုံးဖြတ်ပြီး ဤပြောင်းလဲမှုကို ချိတ်ဆက်ထားသော element များဆီသို့ ပျံ့နှံ့စေရန် လိုအပ်သည်။ ထိုသို့သော ပြောင်းလဲမှုကို 'event' ဟုခေါ်သည်။

Event တစ်ခု ဖြစ်ပေါ်သောအခါ၊ simulator သည် ထို event မှ သက်ရောက်မှုရှိသော circuit element များကိုသာ စစ်ဆေးသည်။ ရလဒ်အနေဖြင့်၊ digital simulator များတွင် matrix analysis မလိုအပ်ပါ။ နှိုင်းယှဉ်ကြည့်ပါက၊ analog simulator များသည် analog components များ၏ forward and reverse transmission properties ကြောင့် ဆားကစ်တစ်ခုလုံး၏ အပြုအမူအတွက် အကြိမ်ကြိမ် ဖြေရှင်းရမည်ဖြစ်သည်။ ဤကွာခြားချက်သည် digital circuit simulator များအတွက် သိသိသာသာ တွက်ချက်မှုဆိုင်ရာ အားသာချက်ကို ဖြစ်ပေါ်စေပြီး၊ ၎င်းသည် digital simulation များ၏ သိသိသာသာ ပိုမိုမြန်ဆန်သော မြန်နှုန်းတွင် ထင်ဟပ်နေသည်။

### ၁.၁.၅ Mixed-Signal သရုပ်ဖော်ခြင်း

ခေတ်သစ် ဆားကစ်များတွင် analog နှင့် digital ဆားကစ်များ ရောနှောပါဝင်လေ့ရှိသည်။ ထိုကဲ့သို့သော ဆားကစ်များကို ထိရောက်တိကျစွာ သရုပ်ဖော်ရန်၊ analog နှင့် digital simulation နည်းစနစ်များ ရောနှောရန် လိုအပ်သည်။ Analog simulation algorithms များကို digital simulation algorithms များနှင့် ပေါင်းစပ်သောအခါ၊ ရလဒ်ကို 'mixed-mode simulation' ဟု ခေါ်ဆိုသည်။

လက်တွေ့တွင် အသုံးပြုသော mixed-mode simulation ကို အကောင်အထည်ဖော်သည့် အခြေခံနည်းလမ်းနှစ်ခုမှာ 'native mode' နှင့် 'glued mode' approaches များဖြစ်ကြသည်။ Native mode simulator များသည် analog algorithm နှင့် digital algorithm နှစ်ခုလုံးကို တူညီသော executable တစ်ခုတည်းတွင် အကောင်အထည်ဖော်သည်။ Glued mode simulator များသည် အမှန်တကယ်တွင် simulator နှစ်ခုကို အသုံးပြုပြီး တစ်ခုမှာ analog ဖြစ်ပြီး နောက်တစ်ခုမှာ digital ဖြစ်သည်။

ဤ simulator အမျိုးအစားသည် executable နှစ်ခု အချင်းချင်း ထိရောက်စွာ ဆက်သွယ်နိုင်ရန်အတွက် input/output protocol တစ်ခုကို သတ်မှတ်ရမည်။ ဆက်သွယ်ရေးဆိုင်ရာ ကန့်သတ်ချက်များသည် simulator တစ်ခုလုံး၏ မြန်နှုန်းနှင့် တစ်ခါတစ်ရံ တိကျမှုကို လျှော့ချလေ့ရှိသည်။ တစ်ဖက်တွင်၊ glued mode simulator ကို အသုံးပြုခြင်းဖြင့် သီးခြား executable များအတွက် တီထွင်ထားသော component models များကို ပြုပြင်မွမ်းမံမှုမရှိဘဲ အသုံးပြုခွင့်ပြုသည်။

Ngspice သည် analog နှင့် event-based simulation နှစ်ခုလုံးကို တူညီသော executable တစ်ခုတည်းတွင် ပေးဆောင်သည့် native mode simulator တစ်ခုဖြစ်သည်။ Ngspice ၏ အခြေခံ algorithms များ (XSPICE နှင့် ၎င်း၏ Code Model Subsystem မှ ဆင်းသက်လာသည်) သည် standard SPICE models အားလုံးကို အသုံးပြုခွင့်၊ အသုံးအများဆုံး analog နှင့် digital functions များကို ကြိုတင်သတ်မှတ်ထားသော စုစည်းမှုတစ်ခု ပေးအပ်ပြီး၊ နောက်ထပ် မော်ဒယ်များ တည်ဆောက်နိုင်သည့် ချဲ့ထွင်နိုင်သော အခြေခံတစ်ခုကို ပေးအပ်သည်။

#### ၁.၁.၅.၁ User-Defined Nodes

Ngspice သည် 'User-Defined Node' အမျိုးအစားများ ဖန်တီးခြင်းကို ထောက်ပံ့ပေးသည်။ User-Defined Node အမျိုးအစားများက သင့်အား voltages၊ currents နှင့် digital states များအပြင် အခြားသော data များကို သယ်ဆောင်ပေးသော node များကို သတ်မှတ်ခွင့်ပြုသည်။ Digital node များကဲ့သို့ပင်၊ User-Defined Nodes များသည် event-driven simulation ကို အသုံးပြုသော်လည်း၊ state value သည် arbitrary data type တစ်ခု ဖြစ်နိုင်သည်။ User-Defined Nodes များ၏ ရိုးရှင်းသော ဥပမာအပလီကေးရှင်းတစ်ခုမှာ digital signal processing filter algorithm တစ်ခု၏ သရုပ်ဖော်ခြင်းဖြစ်သည်။ ဤအပလီကေးရှင်းတွင် node တစ်ခုချင်းစီသည် real သို့မဟုတ် integer value တစ်ခုဟု ယူဆနိုင်သည်။ ပိုမိုရှုပ်ထွေးသော အပလီကေးရှင်းများတွင် digital data vectors သို့မဟုတ် electronic မဟုတ်သော data များပင် ပါဝင်သော complex data ပါဝင်သည့် အမျိုးအစားများကို သတ်မှတ်နိုင်သည်။

Ngspice digital simulation သည် အမှန်တကယ်တွင် ဤ User-Defined Node စွမ်းရည်၏ အထူးကိစ္စရပ်တစ်ခုအဖြစ် အကောင်အထည်ဖော်ထားပြီး၊ digital state ကို Boolean logic state နှင့် strength value တစ်ခုကို သိုလှောင်ထားသော data structure တစ်ခုဖြင့် သတ်မှတ်ပါသည်။

### ၁.၁.၆ Mixed-Level Simulation (Electronic and TCAD)

Ngspice သည် ၎င်း၏ code ကို CIDER (အသေးစိတ်ကို အခန်း ၂၆ တွင်ကြည့်ပါ) နှင့် ပေါင်းစည်းခြင်းဖြင့် mixed-level simulation ကို အကောင်အထည်ဖော်သည်။

CIDER သည် နည်းပညာဆိုင်ရာ parameters များနှင့် ဆားကစ် စွမ်းဆောင်ရည်အကြား တိုက်ရိုက်ချိတ်ဆက်မှုကို ပေးအပ်သော ရောနှောအဆင့် ဆားကစ်နှင့် ပစ္စည်းကိရိယာ simulator တစ်ခုဖြစ်သည်။ ရောနှောအဆင့် ဆားကစ်နှင့် ပစ္စည်းကိရိယာ simulator သည် ဆားကစ်အတွင်းရှိ အရေးပါသော ပစ္စည်းကိရိယာများကို ဂဏန်းနည်းဖြင့် မော်ဒယ်လုပ်ခြင်းဖြင့် သီးသန့် ဆားကစ် သို့မဟုတ် ပစ္စည်းကိရိယာ simulator ထက် ပိုမိုတိကျသော သရုပ်ဖော်မှုကို ပေးစွမ်းနိုင်သည်။ အရေးမကြီးသော ပစ္စည်းကိရိယာများအတွက် Compact model များကို အသုံးပြုနိုင်သည်။

CIDER သည် ngspice ကို အတွင်းပိုင်း C-based device simulator တစ်ခုသို့ ချိတ်ဆက်ပေးကာ၊ ထို့ကြောင့် circuit analyses၊ semiconductor devices အတွက် compact models နှင့် interactive user interface တို့ကို ပေးအပ်သည်။ CIDER သည် Poisson's equation နှင့် electron and hole current-continuity equations တို့၏ အဖြေအပေါ် အခြေခံ၍ တိကျသော၊ တစ်ဖက်မြင် နှင့် နှစ်ဖက်မြင် ဂဏန်းဆိုင်ရာ device models များကို ပေးအပ်သည်။ CIDER သည် Stanford two-dimensional device simulator PISCES [PINT85] တွင် တွေ့ရသော တူညီသည့် အခြေခံ physical models များစွာကို ထည့်သွင်းထားသည်။ CIDER သို့ ထည့်သွင်းမှုမှာ ဆားကစ်နှင့် ၎င်း၏ compact models များ၏ SPICE-like description နှင့် ဂဏန်းနည်းဖြင့် မော်ဒယ်လုပ်ထားသော ပစ္စည်းကိရိယာများ၏ တည်ဆောက်ပုံများ၏ PISCES-like descriptions များ ပါဝင်သည်။ ရလဒ်အနေဖြင့်၊ ဤကိရိယာနှစ်ခုနှင့် ရင်းနှီးပြီးသား ဒီဇိုင်နာများအတွက် CIDER သည် အကျွမ်းတဝင်ရှိသင့်သည်။

CIDER input format သည် ကြီးမားသော ပျော့ပျောင်းမှုရှိပြီး physical model parameters များသို့ ဝင်ရောက်ခွင့် တိုးမြှင့်ခွင့်ပြုသည်။ State-of-the-art devices များကို သရုပ်ဖော်ခွင့်ပြုရန် physical models အသစ်များကို ထည့်သွင်းထားသည်။ ၎င်းတို့တွင် scaled-down MOSFETs များတွင် အရေးပါသော transverse field mobility degradation [GATE90] နှင့် poly-emitter bipolar transistors များအတွက် polysilicon model တို့ပါဝင်သည်။

Temperature dependence ကို \(- 50^{\circ}\mathrm{C}\) မှ \(150^{\circ}\mathrm{C}\) အထိ အကွာအဝေးအတွင်း physical models အများစုအတွက် ထည့်သွင်းထားသည်။ Numerical models များကို တစ်ပိုင်းလျှပ်ကူးပစ္စည်း အမျိုးအစားအားလုံး- resistors၊ MOS capacitors၊ diodes၊ BJTs၊ JFETs နှင့် MOSFETs များကို သရုပ်ဖော်ရန် အသုံးပြုနိုင်သည်။ BJTs နှင့် JFETs များကို substrate contact ပါသည်ဖြစ်စေ၊ မပါသည်ဖြစ်စေ မော်ဒယ်လုပ်နိုင်သည်။ Device internal states များ၏ စီမံခန့်ခွဲမှုအတွက် ထောက်ပံ့မှုကို ထည့်သွင်းထားသည်။ ပစ္စည်းကိရိယာအတွင်းပိုင်း အခြေအနေများ၏ Post-processing ကို ngspice ၏ control language user interface ကို အသုံးပြု၍ လုပ်ဆောင်နိုင်သည်။ ယခင်က တွက်ချက်ထားသော အခြေအနေများကို နောက်ဆက်တွဲ ခွဲခြမ်းစိတ်ဖြာမှုများအတွက် တိကျသော initial guesses များ ပေးအပ်ရန် ပရိုဂရမ်ထဲသို့ load လုပ်နိုင်သည်။

CIDER က အသုံးပြုသော basic semiconductor equations များနှင့် physical models များ၏ အသေးစိတ်ကို ဤလက်စွဲတွင် မဖော်ပြထားပါ။ ကံမကောင်းစွာဖြင့် သက်ဆိုင်ရာ နောက်ခံအကြောင်းအရာအားလုံးကို ဖော်ပြသည့် အခြားသော တစ်ခုတည်းသော ရင်းမြစ်မှာ မရှိပါ။ Device simulation ၏ ပြည့်စုံသော သုံးသပ်ချက်များကို [PINT90] နှင့် [SELB84] စာအုပ်တွင် တွေ့နိုင်သည်။ CODECS (CIDER ၏ ရှေ့ပြေး) နှင့် ၎င်း၏ inversion-layer mobility model ကို [MAYA88] နှင့် [LGATE90] တွင် အသီးသီး ဖော်ပြထားသည်။ PISCES နှင့် ၎င်း၏ models များကို [PINT85] တွင် ဖော်ပြထားသည်။ CIDER က အသုံးပြုသော PISCES models များအတွက် Temperature dependencies များကို [SOLL90] တွင် ရယူနိုင်သည်။

Linux အသုံးပြုသူများအတွက် TCAD software GSS ၏ ngspice နှင့် ပူးပေါင်းဆောင်ရွက်မှုသည် စိတ်ဝင်စားဖွယ်ဖြစ်နိုင်သည်၊ https://ngspice.sourceforge.io/gss.html တွင်ကြည့်ပါ။ သို့သော် ဤပရောဂျက်ကို ဆက်လက်ထိန်းသိမ်းတော့မည်မဟုတ်ဘဲ၊ Genius simulator သို့ ကူးပြောင်းသွားပြီး၊ open source cogenda genius အဖြစ် ရရှိနိုင်ဆဲဖြစ်သည်။

## ၁.၂ ပံ့ပိုးထားသော ခွဲခြမ်းစိတ်ဖြာမှုများ

Ngspice simulator သည် အောက်ပါ ခွဲခြမ်းစိတ်ဖြာမှု အမျိုးအစားများကို ထောက်ပံ့ပေးသည်-

1.  DC Analysis (Operating Point and DC Sweep) (၁၁.၃.၂ နှင့် ၁၁.၃.၅)
2.  AC Small-Signal Analysis (၁၁.၃.၁)
3.  Transient Analysis (၁၁.၃.၁၀)
4.  Pole-Zero Analysis (၁၁.၃.၆)
5.  Small-Signal Distortion Analysis (၁၁.၃.၃)
6.  Sensitivity Analysis (၁၁.၃.၇)
7.  Noise Analysis (၁၁.၃.၄)

ခွဲခြမ်းစိတ်ဖြာမှု အမျိုးအစားများကို အောက်တွင် ဖော်ပြထားပြီး၊ အထက်ပါ cross-references များမှာ ၎င်းတို့ကို တောင်းဆိုရန် အသုံးပြုသည့် netlist directives များဖြစ်သည်။ Analog သီးသန့် အပလီကေးရှင်းများသည် Pole-Zero၊ Distortion၊ Sensitivity နှင့် Noise ခွဲခြမ်းစိတ်ဖြာမှုများကို အကောင်အထည်မဖော်သော Code Model subsystem မှလွဲ၍ analysis mode အားလုံးကို အသုံးပြုနိုင်သည်။ Digital နှင့် User-Defined Node အမျိုးအစားများ ပါဝင်သော Event-driven အပလီကေးရှင်းများသည် DC (operating point and DC sweep) နှင့် Transient ကိုသာ အသုံးပြုနိုင်သည်။

မတူညီသော ခွဲခြမ်းစိတ်ဖြာမှုများနှင့် ngspice ၏ အခြေခံ simulation algorithms နှစ်ခုကြား ဆက်နွယ်မှုကို နားလည်ရန်၊ ခွဲခြမ်းစိတ်ဖြာမှု အမျိုးအစားတစ်ခုချင်းစီက ဘာကိုဆိုလိုသည်ကို နားလည်ရန် အရေးကြီးသည်။ ဤအချက်ကို အောက်တွင် အသေးစိတ်ဖော်ပြထားသည်။

### ၁.၂.၁ DC ခွဲခြမ်းစိတ်ဖြာခြင်း

Ngspice ၏ DC analysis အပိုင်းသည် inductors များကို short လုပ်၍ capacitors များကို open လုပ်ထားသော ဆားကစ်၏ dc operating point ကို ဆုံးဖြတ်သည်။ DC analysis options များကို .DC, .TF, နှင့် .OP control lines များပေါ်တွင် သတ်မှတ်သည်။

DC analysis သည် system description အတွင်းရှိ source များပေါ်တွင် အချိန်အပေါ်မှီခိုမှုကို ထည့်သွင်းစဉ်းစားခြင်းမရှိပါ။ Simulator algorithm သည် ဆားကစ်ကို analog simulator algorithm လိုအပ်သော အပိုင်းများနှင့် event-driven algorithm လိုအပ်သော အပိုင်းများအဖြစ် ပိုင်းခြားသည်။ ထို့နောက် subsystem block တစ်ခုချင်းစီကို analog nodes များနှင့် event-driven nodes များကြားရှိ interfaces များကို system တစ်ခုလုံးအတွင်း ညီညွတ်မှုရှိစေရန် အကြိမ်ကြိမ် ပြုပြင်ကာ အဖြေရှာသည်။

System အတွင်းရှိ node အားလုံးအတွက် stable values များ ရရှိပြီးသည်နှင့်၊ analysis သည် ရပ်တန့်သွားပြီး ရလဒ်များကို သင်တောင်းဆိုသလို ပြသခြင်း သို့မဟုတ် print ထုတ်ခြင်း ပြုလုပ်နိုင်သည်။

dc analysis ကို transient analysis မတိုင်မီ transient initial conditions များကို ဆုံးဖြတ်ရန်၊ နှင့် ac small-signal analysis မတိုင်မီ nonlinear devices များအတွက် linearized, small-signal models များကို ဆုံးဖြတ်ရန် အလိုအလျောက် လုပ်ဆောင်သည်။ တောင်းဆိုပါက၊ transfer function (input source နှင့် output variable အချိုး)၊ input resistance နှင့် output resistance တို့၏ DC small-signal value ကိုလည်း DC solution ၏ တစ်စိတ်တစ်ပိုင်းအဖြစ် တွက်ချက်သည်။ DC analysis ကို DC transfer curves များ ထုတ်လုပ်ရန်လည်း အသုံးပြုနိုင်သည်- သတ်မှတ်ထားသော independent voltage, current source, resistor သို့မဟုတ် temperature ကို user-specified range တစ်ခုအတွင်း step လုပ်ပြီး sequential source value တစ်ခုချင်းစီအတွက် DC output variables များကို သိမ်းဆည်းသည်။

### ၁.၂.၂ AC Small-Signal ခွဲခြမ်းစိတ်ဖြာခြင်း

AC analysis သည် analog nodes များအတွက်သာ ကန့်သတ်ထားပြီး သတ်မှတ်ထားသော ကြိမ်နှုန်းတစ်ခု သို့မဟုတ် ကြိမ်နှုန်းအစုတစ်ခုတွင် ဖော်ပြထားသော analog system ၏ small signal, sinusoidal solution ကို ကိုယ်စားပြုသည်။ ဤ analysis သည် DC analysis နှင့် ဆင်တူပြီး ၎င်းသည် သတ်မှတ်ထားသော stimulus frequencies အစုတစ်ခုတွင် input node တစ်ခုတည်းပါသော ဖော်ပြထားသည့် system ၏ steady-state behavior ကို ကိုယ်စားပြုသောကြောင့်ဖြစ်သည်။

ပရိုဂရမ်သည် ဆားကစ်၏ dc operating point ကို ဦးစွာ တွက်ချက်ပြီး ဆားကစ်အတွင်းရှိ nonlinear devices အားလုံးအတွက် linearized, small-signal models များကို ဆုံးဖြတ်သည်။ ထို့နောက် ရလဒ် linear circuit ကို user-specified frequency range တစ်ခုအတွင်း ခွဲခြမ်းစိတ်ဖြာသည်။ ac small-signal analysis ၏ အလိုရှိသော output သည် အများအားဖြင့် transfer function (voltage gain, transimpedance, etc) ဖြစ်သည်။ ဆားကစ်တွင် ac input တစ်ခုသာရှိပါက၊ output variables များသည် input နှင့်စပ်လျဉ်း၍ output variable ၏ transfer function နှင့် တူညီသောတန်ဖိုးရှိစေရန် ထို input ကို unity နှင့် zero phase အဖြစ် သတ်မှတ်ခြင်းသည် အဆင်ပြေသည်။

### ၁.၂.၃ Transient ခွဲခြမ်းစိတ်ဖြာခြင်း

Transient analysis သည် DC analysis ကို time domain သို့ တိုးချဲ့ခြင်းဖြစ်သည်။ Transient analysis တစ်ခုသည် time-varying behavior ကို စတင်သရုပ်ဖော်ရန်အတွက် point of departure တစ်ခု ပေးအပ်ရန် DC solution တစ်ခုကို ဦးစွာ ရယူသည်။ DC solution ရရှိပြီးသည်နှင့်၊ system ၏ time-dependent aspects များကို ပြန်လည်ထည့်သွင်းပြီး၊ simulator algorithms နှစ်ခုသည် system တစ်ခုလုံး၏ time varying behavior အတွက် တဖြည်းဖြည်းချင်း အဖြေရှာသည်။ Node values များတွင် inconsistencies များကို simulation algorithms နှစ်ခုက ဖြေရှင်းပေးသောကြောင့် analysis မှ ဖန်တီးသော time-dependent waveforms များသည် သတ်မှတ်ထားသော simulated time interval တစ်ခုလုံးအတွင်း consistent ဖြစ်သည်။ သတ်မှတ်ထားသော time interval အတွက် node behavior ၏ ရလဒ် time-varying descriptions များကို သင် ရယူနိုင်သည်။

Time dependent မဟုတ်သော source အားလုံး (ဥပမာ power supplies) ကို ၎င်းတို့၏ dc value အတိုင်း သတ်မှတ်သည်။ Transient time interval ကို .TRAN control line ပေါ်တွင် သတ်မှတ်သည်။

### ၁.၂.၄ Pole-Zero ခွဲခြမ်းစိတ်ဖြာခြင်း

Ngspice တွင် pole-zero analysis သည် small-signal ac transfer function အတွင်းရှိ poles နှင့်/သို့မဟုတ် zeros များကို တွက်ချက်သည်။ Ngspice သည် dc operating point ကို ဦးစွာ တွက်ချက်ပြီး ဆားကစ်အတွင်းရှိ nonlinear devices အားလုံးအတွက် linearized, small-signal models ကို ဆုံးဖြတ်သည်။ ထို့နောက် transfer function ၏ poles နှင့် zeros များကို ရှာဖွေရန် small-signal circuit model ကို အသုံးပြုသည်။ Transfer function အမျိုးအစားနှစ်မျိုးကို ခွင့်ပြုသည်- တစ်မျိုးမှာ (output voltage)/(input voltage) ပုံစံဖြစ်ပြီး နောက်တစ်မျိုးမှာ (output voltage)/(input current) ပုံစံဖြစ်သည်။ ဤ transfer function အမျိုးအစားနှစ်မျိုးသည် cases အားလုံးကို လွှမ်းခြုံပြီး input/output impedance နှင့် voltage gain ကဲ့သို့သော function များ၏ poles/zeros များကို ရှာဖွေနိုင်သည်။ input နှင့် output ports များကို node နှစ်စုံအဖြစ် သတ်မှတ်သည်။ Pole-zero analysis သည် resistors, capacitors, inductors, linear-controlled sources, independent sources, BJTs, MOSFETs, JFETs နှင့် diodes များဖြင့် အလုပ်လုပ်သည်။ Transmission lines များကို မထောက်ပံ့ပါ။

ဤ analysis တွင် အသုံးပြုသော နည်းလမ်းမှာ sub-optimal numerical search တစ်ခုဖြစ်သည်။ ကြီးမားသော ဆားကစ်များအတွက် poles နှင့် zeros အားလုံးကို ရှာဖွေရန် အချိန်အတော်အတန် ကြာနိုင်သည် သို့မဟုတ် ပျက်ကွက်နိုင်သည်။ ကျေးဇူးပြု၍ သတိပြုပါ၊ အချို့သော ဆားကစ်များအတွက် နည်းလမ်းသည် "lost" ဖြစ်သွားပြီး poles သို့မဟုတ် zeros အရေအတွက် အလွန်အကျွံ ရှာတွေ့နိုင်သည်။

### ၁.၂.၅ Small-Signal Distortion ခွဲခြမ်းစိတ်ဖြာခြင်း

Ngspice တွင် distortion analysis သည် small input signal magnitudes များအတွက် steady-state harmonic and intermodulation products များကို တွက်ချက်သည်။ ဆားကစ်သို့ input အဖြစ် single frequency ၏ signals များကို သတ်မှတ်ပါက၊ ဆားကစ်အတွင်းရှိ အမှတ်တိုင်းတွင် second and third harmonics များ၏ complex values များကို ဆုံးဖြတ်သည်။ ဆားကစ်သို့ frequency နှစ်ခု၏ signals များ input လုပ်ပါက၊ analysis က input frequencies များ၏ sum and difference နှင့် larger frequency ၏ second harmonic မှ smaller frequency ၏ difference တို့တွင် circuit variables များ၏ complex values များကို ရှာဖွေသည်။ Distortion analysis ကို အောက်ပါ nonlinear devices များအတွက် ထောက်ပံ့သည်-

-   Diodes (DIO),- BJT,- JFET (level 1),- MOSFETs (levels 1, 2, 3, 9, and BSIM1),- MESFET (level 1)။

Linear devices အားလုံးကို distortion analysis က အလိုအလျောက် ထောက်ပံ့သည်။ ဆားကစ်ထဲတွင် switches များ ရှိနေပါက၊ distortion calculations အတွက် အသုံးပြုသော small excitations များအောက်တွင် switches များ state မပြောင်းလဲသရွေ့ analysis သည် ဆက်လက်တိကျနေပါသည်။

Device model တစ်ခုက direct small signal distortion analysis ကို မထောက်ပံ့ပါက၊ Fourier သို့မဟုတ် FFT statements များကို အသုံးပြု၍ scripting ဖြင့် output ကို အကဲဖြတ်ပါ။

### ၁.၂.၆ Sensitivity ခွဲခြမ်းစိတ်ဖြာခြင်း

Ngspice သည် circuit variables အားလုံး (model parameters အပါအဝင်) နှင့် စပ်လျဉ်း၍ output variable တစ်ခု၏ DC operating-point sensitivity သို့မဟုတ် AC small-signal sensitivity ကို တွက်ချက်နိုင်သည်။ Ngspice သည် device တစ်ခုချင်းစီ၏ parameter တစ်ခုချင်းစီကို သီးခြားစီ perturb လုပ်ခြင်းဖြင့် output variable (node voltage သို့မဟုတ် branch current) တစ်ခု၏ ခြားနားချက်ကို တွက်ချက်သည်။

### ၁.၂.၇ Noise ခွဲခြမ်းစိတ်ဖြာခြင်း

Ngspice တွင် noise analysis သည် ပေးထားသော ဆားကစ်တစ်ခုအတွက် device-generated noise ကို တိုင်းတာသည်။ input source တစ်ခုနှင့် output port တစ်ခုကို ပေးအပ်သောအခါ၊ analysis သည် output port တွင် voltage အဖြစ် တိုင်းတာသည့်အတိုင်း device တစ်ခုချင်းစီနှင့် device တစ်ခုအတွင်းရှိ noise generator တစ်ခုချင်းစီ၏ noise contributions များကို တွက်ချက်သည်။ Noise analysis သည် output noise ကို အခြေခံ၍ ဆားကစ်၏ equivalent input noise ကိုလည်း တွက်ချက်သည်။ ဤသို့ပြုလုပ်ခြင်းကို specified range တစ်ခုအတွင်းရှိ frequency point တိုင်းအတွက် ပြုလုပ်သည် - noise ၏ တွက်ချက်ထားသော value သည် stationary Gaussian stochastic process အဖြစ် ရှုမြင်ထားသော circuit variable ၏ spectral density နှင့် ကိုက်ညီသည်။ Spectral densities များကို တွက်ချက်ပြီးနောက်၊ noise analysis သည် ဤ frequency range အတွင်း total noise voltage and current ကို ရောက်ရှိရန် specified frequency range အတွင်း ဤ values များကို integrate လုပ်သည်။ တွက်ချက်ထားသော values များသည် stationary Gaussian processes များအဖြစ် ရှုမြင်ထားသော circuit variables ၏ variance နှင့် ကိုက်ညီသည်။

Temperature သည် ngspice တွင် analysis option တစ်ခုထက် ဆားကစ်တစ်ခုလုံးနှင့် ဆက်စပ်နေသော property တစ်ခုဖြစ်သည်။ Circuit temperature တွင် default (nominal) value အနေဖြင့် \(27^{\circ}C\) (300.15 K) ရှိပြီး ၎င်းကို .option control line (၁၁.၁.၁ ကိုကြည့်ပါ) ရှိ TEMP option ကို အသုံးပြု၍ သို့မဟုတ် .option TEMP line ထက် ဦးစားပေးသည့် .TEMP line (၂.၁၄ ကိုကြည့်ပါ) ဖြင့် ပြောင်းလဲနိုင်သည်။ ထို့ကြောင့် ခွဲခြမ်းစိတ်ဖြာမှုအားလုံးကို circuit temperature တွင် လုပ်ဆောင်ပြီး၊ မတူညီသော အပူချိန်များတွင် ဆားကစ်၏ အပြုအမူကို သရုပ်ဖော်လိုပါက အပူချိန်တစ်ခုချင်းစီအတွက် netlist တစ်ခု ပြင်ဆင်သင့်သည်။

Ngspice အတွက် input data အားလုံးကို circuit nominal temperature တွင် တိုင်းတာထားသည်ဟု ယူဆသည်။ ဤတန်ဖိုးကို .model ပေါ်ရှိ TNOM parameter ကို သတ်မှတ်ခြင်းဖြင့် temperature effects များကို မော်ဒယ်လုပ်သော မည်သည့် device အတွက်မဆို ထပ်မံ၍ override လုပ်နိုင်သည်။ Individual instances များသည် instance ပေါ်ရှိ TEMP နှင့် DTEMP parameters များ၏ specification မှတစ်ဆင့် circuit temperature ကို ထပ်မံ၍ override လုပ်နိုင်သည်။

Temperature dependent support ကို voltage and current sources (independent and controlled) နှင့် BSIM models များမှလွဲ၍ devices အားလုံးအတွက် ထောက်ပံ့ပေးထားသည်။ BSIM MOSFETs များတွင် ngspice သို့ input မလုပ်မီ model parameters အားလုံးကို ချိန်ညှိပေးသော alternate temperature dependency scheme ရှိသည်။

BSIM temperature adjustment ၏အသေးစိတ်အတွက် [6] နှင့် [7] ကိုကြည့်ပါ။ Temperature သည် BJT နှင့် diode model equations များ၏ exponential terms များတွင် အတိအလင်း ပေါ်လာသည်။ ထို့အပြင် saturation currents များတွင် built-in temperature dependence ရှိသည်။ BJT models များတွင် saturation current ၏ temperature dependence ကို အောက်ပါအတိုင်း ဆုံးဖြတ်သည်-

\[I_{S}(T_{1}) = I_{S}(T_{0})\left(\frac{T_{1}}{T_{0}}\right)^{XTI}\exp \left(\frac{E_{g}q(T_{1} - T_{0})}{k(T_{1}T_{0})}\right) \quad (1.1)\]

ဤနေရာတွင် \(k\) သည် Boltzmann's constant၊ \(q\) သည် electronic charge၊ \(E_{g}\) သည် energy gap model parameter နှင့် \(XTI\) သည် saturation current temperature exponent (model parameter တစ်ခုလည်းဖြစ်ပြီး များသောအားဖြင့် 3 နှင့် ညီသည်) တို့ဖြစ်သည်။

Forward and reverse beta ၏ temperature dependence သည် အောက်ပါ ဖော်မြူလာအတိုင်းဖြစ်သည်-

\[B(T_{1}) = B(T_{0})\left(\frac{T_{1}}{T_{0}}\right)^{XTB} \quad (1.2)\]

ဤနေရာတွင် \(T_{0}\) နှင့် \(T_{1}\) တို့သည် degrees Kelvin ဖြစ်ပြီး၊ \(XTB\) သည် user-supplied model parameter ဖြစ်သည်။ Beta အပေါ် temperature effects များကို \(B_{F}\) , \(I_{SE}\) , \(B_{R}\) , နှင့် \(I_{SC}\) (အသီးသီး SPICE model parameters BF, ISE, BR, and ISC) တို့၏တန်ဖိုးများသို့ သင့်လျော်သော ချိန်ညှိမှုများဖြင့် ဆောင်ရွက်သည်။

Junction diode model တွင် saturation current ၏ Temperature dependence ကို အောက်ပါအတိုင်း ဆုံးဖြတ်သည်-

\[I_{S}(T_{1}) = I_{S}(T_{0})\left(\frac{T_{1}}{T_{0}}\right)^{\frac{XTI}{N}}\exp \left(\frac{E_{g}q(T_{1} - T_{0})}{Nk(T_{1}T_{0})}\right) \quad (1.3)\]

ဤနေရာတွင် \(N\) သည် emission coefficient model parameter ဖြစ်ပြီး၊ အခြား သင်္ကေတများမှာ အထက်ပါအတိုင်း တူညီသော အဓိပ္ပါယ်ရှိသည်။ Schottky barrier diodes များအတွက် saturation current temperature exponent, \(XTI\) ၏တန်ဖိုးသည် များသောအားဖြင့် 2 ဖြစ်သည်ကို သတိပြုပါ။ Temperature သည် device models အားလုံးအတွက် junction potential, U (Ngspice တွင် PHI) ၏တန်ဖိုးတွင် အတိအလင်း ပေါ်လာသည်။

Temperature dependence ကို အောက်ပါအတိုင်း ဆုံးဖြတ်သည်-

\[U(T) = \frac{kT}{q}\ln \left(\frac{N_{a}N_{d}}{N_{i}(T)^{2}}\right) \quad (1.4)\]

========== 44 ==========

ဤနေရာတွင် \(k\) သည် Boltzmann's constant၊ \(q\) သည် electronic charge၊ \(N_{a}\) သည် acceptor impurity density၊ \(N_{d}\) သည် donor impurity density၊ \(N_{i}\) သည် intrinsic carrier concentration နှင့် \(E_{g}\) သည် energy gap တို့ဖြစ်သည်။ Temperature သည် MOSFET model အတွက် surface mobility, \(M_{0}\) (သို့မဟုတ် \(U_{0}\) ) ၏တန်ဖိုးတွင် အတိအလင်း ပေါ်လာသည်။

Temperature dependence ကို အောက်ပါအတိုင်း ဆုံးဖြတ်သည်-

\[M_{0}(T) = \frac{M_{0}(T_{0})}{\left(\frac{T}{T_{0}}\right)^{1.5}} \quad (1.5)\]

Resistors, capacitor နှင့် inductors များအပေါ် temperature ၏သက်ရောက်မှုများကို အောက်ပါဖော်မြူလာဖြင့် မော်ဒယ်လုပ်သည်-

\[R(T) = R(T_{0})\left[1 + TC_{1}(T - T_{0}) + TC_{2}(T - T_{0})^{2}\right] \quad (1.6)\]

ဤနေရာတွင် \(T\) သည် circuit temperature၊ \(T_{0}\) သည် nominal temperature၊ နှင့် \(TC_{1}\) နှင့် \(TC_{2}\) တို့သည် first and second order temperature coefficients များဖြစ်သည်။

## ၁.၄ Convergence (ဆုံရှုံးမှု)

Ngspice သည် circuit description မှ ပေါ်ပေါက်လာသော nonlinear equations များကို ဖြေရှင်းရန် Newton-Raphson algorithm ကို အသုံးပြုသည်။ NR algorithm သည် interactive ဖြစ်ပြီး အောက်ပါအခြေအနေနှစ်ခုစလုံး ကိုင်ထားသည့်အခါ ရပ်ဆိုင်းသည်-

1.  Nonlinear branch currents များသည် tolerance တစ်ခုဖြစ်သော \(0.1\%\) သို့မဟုတ် 1 picoamp (1.0e-12 Amp)၊ မည်သည်က ပိုကြီးသည်ဖြစ်စေ၊ အတွင်းသို့ converge ဖြစ်သည်။
2.  Node voltages များသည် tolerance တစ်ခုဖြစ်သော \(0.1\%\) သို့မဟုတ် 1 microvolt (1.0e-6 Volt)၊ မည်သည်က ပိုကြီးသည်ဖြစ်စေ၊ အတွင်းသို့ converge ဖြစ်သည်။

### ၁.၄.၁ Voltage convergence စံနှုန်း

Algorithm သည် နောက်ဆုံး iteration \(k\) နှင့် လက်ရှိ \((k + 1)\) အကြား ခြားနားချက်သည် အောက်ပါအခြေအနေသို့ ရောက်ရှိသောအခါ convergence သို့ ရောက်ရှိသည်-

\[\left|v_{n}^{(k + 1)} - v_{n}^{(k)}\right|\leq \mathrm{RELTOI} v_{n_{\max}} + \mathrm{VNTOL}, \quad (1.7)\]

ဤနေရာတွင်

\[v_{n_{\max}} = \max \left(\left|v_{n}^{(k + 1)}\right|,\left|v_{n}^{(k)}\right|\right). \quad (1.8)\]

RELTOL (RELative TOLerance) parameter သည် ပုံသေတန်ဖိုး \(10^{- 3}\) ဖြစ်ပြီး၊ node voltage နှင့် ဆက်စပ်၍ solution update သည် မည်မျှသေးငယ်ရမည်ကို သတ်မှတ်ပေးကာ solution သည် converge ဖြစ်ပြီဟု ယူဆရန် ဖြစ်သည်။ VNTOL (absolute convergence) parameter သည် \(1\mu V\) ကို ပုံသေတန်ဖိုးအဖြစ် ထားရှိပြီး၊ node voltages များသည် သုညနီးပါးတန်ဖိုးများ ရှိသောအခါတွင် အရေးပါလာသည်။ ထိုသို့သောကိစ္စတွင် relative parameter တစ်ခုတည်းက အလွန်အမင်း တင်းကျပ်သော tolerances များ လိုအပ်မည်ဖြစ်ပြီး၊ computer round-off error ထက်ပင် နိမ့်ကျနိုင်ကာ ထို့ကြောင့် convergence ကို ဘယ်သောအခါမျှ မရရှိနိုင်ပေ။ VNTOL သည် algorithm အား solution update သည် ၎င်း၏တန်ဖိုးထက် နိမ့်သော မည်သည့် node ကိုမဆို converged အဖြစ် သတ်မှတ်ရန် တွန်းအားပေးသည်။

### ၁.၄.၂ Current convergence စံနှုန်း

Ngspice သည် circuit elements များရှိ non-linear branches များကို ဖော်ပြသည့် non-linear functions များပေါ်တွင် convergence ကို စစ်ဆေးသည်။ Semiconductor devices များတွင် functions များသည် devices များမှတစ်ဆင့် currents များကို ဖော်ပြသောကြောင့် စံနှုန်း၏ အမည်ဖြစ်လာသည်။

Ngspice သည် နောက်ဆုံး voltage အတွက် တွက်ချက်ထားသော nonlinear function ၏တန်ဖိုးနှင့် actual voltage ဖြင့် တွက်ချက်ထားသော တူညီသော current ၏ linear approximation အကြား ခြားနားချက်ကို တွက်ချက်သည်-

\[\left|\vec{i}_{branch}^{(k + 1)} - \vec{i}_{branch}^{(k)}\right|\leq \mathrm{RELTOLL}i_{b_{r m a x}} + \mathrm{ABSTOL}, \quad (1.9)\]

ဤနေရာတွင်

\[i_{b_{r m a x}} = \max \left(\vec{i}_{b r a n c h}^{(k + 1)}\vec{i}_{b r a n c h}^{(k)}\right). \quad (1.10)\]

အထက်ပါ expression နှစ်ခုတွင် \(\vec{i}_{branch}\) သည် current ၏ linear approximation ကို ညွှန်ပြသည်။

### ၁.၄.၃ Convergence ပျက်ကွက်ခြင်း

Ngspice တွင် အသုံးပြုသော algorithm သည် အလွန်ယုံကြည်စိတ်ချရသည်ဟု တွေ့ရှိရသော်လည်း၊ အချို့သော ကိစ္စများတွင် solution တစ်ခုသို့ converge ဖြစ်ရန် ပျက်ကွက်သည်။ ဤသို့ပျက်ကွက်သောအခါ၊ ပရိုဂရမ်သည် job ကို ရပ်ဆိုင်းသည်။ dc analysis တွင် converge ဖြစ်ရန် ပျက်ကွက်ခြင်းသည် အများအားဖြင့် circuit connections၊ element values သို့မဟုတ် model parameter values များကို သတ်မှတ်ရာတွင် error တစ်ခုကြောင့် ဖြစ်သည်။ Regenerative switching circuits သို့မဟုတ် positive feedback ပါသော ဆားကစ်များသည် feedback path ရှိ devices အချို့အတွက် OFF option ကို မသုံးပါက သို့မဟုတ် ဆားကစ်ကို အလိုရှိသော state သို့ converge ဖြစ်စေရန် .nodeset control line ကို မသုံးပါက dc analysis တွင် converge ဖြစ်လိမ့်မည် မဟုတ်ပေ။
